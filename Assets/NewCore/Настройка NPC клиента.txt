Настройка NPC-клиента (у двери → стойка админа → взаимодействие)

Чтобы клиент обходил стены, нужен NavMesh (см. файл «Как запечь NavMesh чтобы NPC не шёл сквозь стены»).

1. Точка у стойки админа
   Создай пустой объект (ПКМ в Hierarchy → Create Empty).
   Назови, например, "AdminCounterSpot".
   Поставь его там, где должен стоять клиент перед стойкой (перед тобой).

2. NPC у двери
   Поставь в сцену модель/префаб клиента (или примитив Capsule) у двери.
   На этот объект добавь:
   - Collider (Capsule Collider или Box — чтобы в него попадал луч взаимодействия).
   - Nav Mesh Agent (Add Component → Nav Mesh Agent).
   - Client NPC (Add Component → NewCore → Client NPC).

3. В компоненте Client NPC
   - Doors — массив дверей. Size = 2, перетащи обе створки (объекты с Interactable Door). Клиент пойдёт к стойке только когда ОБЕ открыты. Если массив пустой — идёт сразу.
   - Counter Target — перетащи AdminCounterSpot (пустой объект у стойки).
   - Arrive Distance — 0.6 (дистанция «пришёл»).
   - Move Speed — 2.5 (скорость ходьбы; задаётся и в Nav Mesh Agent).

4. Проверка
   Запусти сцену. Клиент стоит у двери. Открой дверь (E) — он пойдёт к стойке. Подойди, наведи прицел на клиента, нажми E — вызовется OnInteract().
